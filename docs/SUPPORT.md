# Support

## Routing

| Need | Route |
|---|---|
| Reproducible bug | private owner-controlled issue or task |
| Evidence or claim dispute | update `SOVEREIGN_PRD.md` or `SOURCE_BOUNDARY.md` with evidence |
| Feature request | defer until `package_boundary_earned` is no longer `UNTESTED` |
| General question | owner-controlled private route |
| Security issue | Private route in `../SECURITY.md` |

## What To Include

- repo version, branch, or commit
- operating context
- exact artifact or file path involved
- expected behavior versus observed behavior
- any owner-held dependency or limitation already known

## Response Expectations

| Request Type | Expected Handling |
|---|---|
| Bug | Triage against the extraction-boundary state before changing code. |
| Question | Answer from repo-local truth surfaces or mark unresolved. |
| Evidence dispute | Update the authority metric only with artifact evidence. |
| Feature request | Reject or defer unless it advances D-06, Phase 02b, Phase 03, or operational coherence. |

## Boundary Note

Support replies do not change repo truth by themselves. Public claims change
only when the evidence and docs surface are updated.
